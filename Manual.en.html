01. Before you start

What VoiceBridge does

VoiceBridge converts documents to audio, transcribes audio/video, creates subtitles, and supports small manual cleanup operations on audio and video.

The app is organized by workflow. Usually you select a section, load a file, check the essential options, and start the job.

Operational requirements

  • Edge TTS requires internet.
  • Local TTS requires a local XTTS-v2 model.
  • Transcription requires Whisper large-v3.
  • SRT output also requires the alignment model for the spoken language.
  • Audio/Video Cleanup use ffmpeg.
  • Video glitch detection uses OpenCV.

Optional prerequisites

ComponentWhen it is needed
Microsoft WordOnly for old .doc documents.
Tesseract OCROnly for scanned or image-based PDFs.
CUDAOptional. If unavailable, use Auto or CPU.
The practical minimum resolution is Full HD, 1920x1080. Smaller screens or high display scaling may make some screens harder to use.

02. Home and sidebar

Home shows the operational status of the app: Edge TTS, local runtime, STT models, Local TTS, DVAE, and video tools. If something is missing, the affected section shows the required message or action button.

Main sections

  • Text to Speech: generates MP3 files from documents.
  • Local Voices: manages profiles, datasets, setup, and training.
  • Transcription: creates Markdown transcripts or SRT subtitles.
  • Subtitles: embeds or burns SRT subtitles into videos.
  • Audio Cleanup: fixes small audio defects.
  • Video Cleanup: fixes individual problematic frames.
The interface language is changed from the sidebar. File and output fields start empty at every launch to reduce the risk of working on the wrong file.

03. Text to Speech

  1. Open Text to Speech.
  2. Select a .txt, .docx, .doc, or .pdf file.
  3. Choose engine, voice, language, and speed.
  4. Choose the MP3 output path.
  5. Press Generate MP3.
  6. When the job finishes, use Open output, Open folder, or Open in Audio Cleanup.

Edge TTS

Edge TTS uses Microsoft online voices. If internet is unavailable, VoiceBridge disables Edge TTS, temporarily switches to Local TTS when possible, and retries the voice-list check in the background.

Local TTS

Local TTS uses XTTS-v2 in the local runtime. Before using it, download the model with Download XTTS-v2, then choose the local voice profile, preset, and device.

PresetRecommended use
StableMore conservative, useful for long tests.
BalancedRecommended compromise.
NaturalMore expressive, potentially less stable.

Multi-voice blocks

Splits the text into blocks and lets you assign different voices and speeds. The output remains one final MP3. When available, VoiceBridge also saves a JSON timeline next to the MP3.

04. Local Voices

Local Voices contains Profiles, Datasets, Setup, and Training.

Profiles

  1. Create a Reference clone profile if you already have authorized reference audio.
  2. Create a Modeling dataset profile if you want to collect clips for future training.
  3. To record reference audio, use Record.
  4. Listen to the cleaned WAV and choose whether to keep it, retry, or cancel.
The profile type is selected during creation and then locked. If you delete a profile with linked modeling work, the app asks for destructive confirmation before deleting the dataset and safely identifiable assets.

05. Local Voices > Datasets

  1. Create a Modeling dataset profile first.
  2. Open Local Voices > Datasets.
  3. Use Generate guided text to create a local prompt suitable for the dataset language.
  4. Use Record from text to record the guided clip.
  5. Listen to the result, then keep it or retry.
  6. Fix transcripts when needed and use Verify text if the check must be rerun.
  7. Use Exclude export to keep a clip in the dataset but skip it during export.

Clip states

Ready usable audio and transcript.

Checking text verification is running.

Match OK audio and text are consistent.

Needs review text should be checked.

Check error verification did not complete.

Export dataset is available only when the dataset reaches the minimum readiness level. Export copies only ready clips and does not modify the working dataset.
Guided prompts generated by the app are limited to 200 characters to stay compatible with XTTS-v2 fine-tuning. If you paste or load custom text, keep each clip within the same limit: exports with longer transcript rows are blocked during training preparation to avoid trainer errors.

06. Setup and Training

Setup

  1. Export a dataset first.
  2. Choose the export from the dropdown.
  3. Set output folder, device, epochs, and batch size.
  4. Select a checkpoint only when resuming training.
  5. Download training assets if they are missing.
  6. Run Refresh preflight.
  7. Save with Save training config.

Training

  1. Select a configured job.
  2. Use Prepare.
  3. Use Dry run to verify without training.
  4. Use Start training.
  5. Use Cancel to terminate the external process.
  6. Use Open job folder to inspect logs and output.
Training asset downloads can be cancelled with Cancel download. If CUDA fails, the app may offer to switch the job to CPU and retry.
Buttons follow the job state: after Save training config, only Prepare is available; after Prepare, only Dry run is available; after a successful Dry run, Start training becomes available. If the job fails or is cancelled, start again from Prepare.

07. Transcription

  1. Open Transcription.
  2. If Download Whisper large-v3 appears, download the model first.
  3. Select audio or video.
  4. Choose the mode: Markdown, automatic SRT, or SRT from provided text.
  5. Choose language and device.
  6. Start the job.
ModeWhen to use it
Transcript Markdown (.md)Readable transcription without SRT timing.
Auto subtitles (.srt)Automatically timed subtitles.
Subtitles from provided text (.srt)You already have text and want to align it to audio/video.
SRT modes require the alignment model for the spoken language. If it is missing, the app asks whether to download it.

08. Subtitles

  1. Select the source video.
  2. Select the .srt file.
  3. Choose the output path.
  4. Use Embed SRT track for a selectable subtitle track.
  5. Use Burn in SRT to make subtitles part of the video image.
  6. For burn-in, adjust position, size, outline, shadow, colors, and quality.

Burn-in re-encodes the video. Lower CRF modes increase quality and file size.

09. Audio Cleanup

  1. Select a supported audio file.
  2. Use the waveform, zoom, or Start/End fields to select a range.
  3. If a TTS timeline exists, use TTS blocks.
  4. Press Cut, Silence, or Fade.
  5. Repeat for multiple corrections.
  6. Check Applied changes.
  7. Press Clean audio.
Changes cannot overlap. Intermediate stages use a lossless format; audio is encoded into the final format only during the last step.

10. Video Cleanup

  1. Select the video.
  2. Choose output and quality.
  3. Use Frame review to load the filmstrip.
  4. Use the arrows near the slider to move one frame at a time.
  5. Use Detect black frames to find isolated black frames.
  6. Use Detect frame glitches to find suspicious non-black frames.
  7. Double-click a result to jump to that frame.
  8. Manually mark frames to fix when needed.
  9. Apply Freeze previous frame or Remove selected frames.
  10. Check Applied changes and press Clean video.
Detect black frames auto-marks only isolated black frames considered repairable. Suspicious non-black frames are only reported and must be reviewed manually.
ActionEffect
Freeze previous frameReplaces the marked frame with the previous frame and keeps the original duration.
Remove selected framesRemoves frames and matching micro-portions of audio, shortening the timeline slightly.

Quick troubleshooting

  • Edge TTS unavailable: check internet or use Local TTS.
  • Local TTS unavailable: download XTTS-v2 and verify the voice profile.
  • Transcription not ready: download Whisper large-v3.
  • SRT not generated: check language and alignment model.
  • Audio/video cleanup fails: check output path, disk space, and files opened by other programs.
  • CUDA unavailable: use Auto or CPU.
  • PDF empty or scanned: install Tesseract OCR and use OCR when requested.
